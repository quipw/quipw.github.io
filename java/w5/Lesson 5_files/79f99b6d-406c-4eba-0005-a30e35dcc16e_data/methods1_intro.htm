Introduction to Methods

What is a method?

A method is a set of instructions that are grouped together to perform some function. The statements execute as a unit. You have encountered a few methods. Can you name any?

  • println method of the PrintStream class as in System.out.println()
  • showMessageDialog method as in JOptionPane.showMessageDialog()
  • showInputDialog method as in JOptionPane.showInputDialog
  • nextInt method of the Scanner class
  • the main method

Your code causes a method to execute by using its name followed by parentheses. The method call may or may not specify parameters in the parentheses.

int age = myScanner.nextInt(); //no parameters

System.out.println("I love Java"); //a String parameter

 

What happens when you call a method?

What goes on behind the scenes when you call System.out.println in your code? A number of statements are executed to display the specified data on the console window. The program flow actually leaves your code and executes the statements in the println method located in PrintStream class in the Java libraries. Return values

Methods may or may not return a value to the calling program. System.out.println() does not return a value.

Scanner's nextInt() method returns an int value. This value essentially replaces the method call in the code.

age = myScanner.nextInt();

If the user enters 42, then 42 replaces myScanner.nextInt() and is assigned to age (put in the box named age).
