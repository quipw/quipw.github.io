Multiple Decisions

 

Sometimes we need to choose among more than two alternatives. For example, we might have a month number and want to print the month name. There are two ways to do this. First we will talk about the if/else if construction.

 

if/else if

The if/else if construction allows us to make multiple tests. If the first test is false, the next in the series is executed.

Look at this code. 

import java.util.*;

/**
   Shows the use of multiple if/ else if statement
   @author Kathleen O'Brien
   @version 1.1
*/
public class MultiIfDemo
{
   public static void main(String[] arg)
   {
      //make a console reader
      Scanner console = new Scanner(System.in);

      //get input
      System.out.print("Enter the month number: ");
      int month = console.nextInt();

      String monthName = "";  //initialize to empty string

      //test the month number
      if (month == 1)
      {
         monthName = "January";
      }
      else if (month == 2)
      {
         monthName = "February";
      }
      else if (month == 3)
      {
         monthName = "March";
      }
      else if (month == 4)
      {
         monthName = "April";
      }
      else if (month == 5)
      {
         monthName = "May";
      }

      else if (month == 6)
      {
         monthName = "June";
      }
      else if (month == 7)
      {
         monthName = "July";
      }
      else if (month == 8)
      {
         monthName = "August";
      }
      else if (month == 9)
      {
         monthName = "September";
      }
      else if (month == 10)
      {
         monthName = "October";
      }

      else if (month == 11)
      {
         monthName = "November";
      }
      else if (month == 12)
      {
         monthName = "December";
      }
      else
      {
         monthName = "Bad Number";
      }

      System.out.println("Month number "+ month + " is " + monthName);
   }//end main method
}//end class

Use the if /else if construction rather than a series of plain ifs. That will make your program more efficient. Usually the last statement will be an else - not an else if. It tells Java what to do if none of the previous tests were true. It is the default action.

Try It Yourself

You can copy, paste into a text editor, save, compile, and execute it yourself. Try several different numbers. Does the code work? Be sure to test 0 and 13. Why those numbers? Because they are the values just outside the valid range of 1 to 12. This is called testing boundary conditions, and it is always a good idea.

Using else if or else?

Do not use else if when the test will always be true. Use else instead. Here is an example of incorrect use of else if as the last statement in a series of decisions

/*assume an int variable called input has already 
been declared and assigned a value
*/
if (input > 0)
{
  System.out.println(input + " is greater than 0");
}
else if (input < 0 ) //correct use of else if
{
  System.out.println(input + " is less than 0");
}
else if (input == 0) //incorrect use of else if - use else
{
  System.out.println(input + " is exactly 0");
}

Look at the last else if. (input ==0) will always be true at this point in the code. If this statement is being executed, input HAS to be 0. We already determined input is not greater than 0 (first test) or less than 0 (second test). The only other posiblitiy is that input is equal to 0. So use else.

The code should look like the following segment. Notice that last statement simply uses else - not else (input == 0) 

/*assume an int variable called input has already 
been declared and assigned a value
*/
if (input > 0)
{
  System.out.println(input + " is greater than 0");
}
else if (input < 0 ) //correct use of else if
{
  System.out.println(input + " is less than 0");
}
else  //correct use of else 
{
  System.out.println(input + " is exactly 0");
}

 

switch

There is another construction we can use to accomplish this task. The switch statement.

The syntax for the switch statement:

switch (n)   //n is already defined and is an integer type
{
   case 1:
      break;
   case 3:
      break;
   default:
}

List as many cases as you need and end each with a break. If you don't put in the break, you "fall through" to the next case and keep executing until you come to a break. This is usually an error.

You can provide a default case that is executed if no case value matches the switch variable.

 

switch Example:

Here is the month name problem using a switch statement.

import java.util.Scanner;

/**
   Shows the use of switch statement
   @author Kathleen O'Brien
   @version 1.1
*/
public class MonthSwitch
{
   public static void main(String[] arg)
   {
      //make a console reader
      Scanner console = new Scanner(System.in);

      //get input
      System.out.print("Enter the month number: ");
      int month = console.nextInt();

      String monthName = "";  //initialize to empty string

      //test the month number
      switch (month)
      {
         case 1:
            monthName = "January";
            break;
         case 2:
            monthName = "February";
            break;
         case 3:
            monthName = "March";
            break;
         case 4:
            monthName = "April";
            break;
         case 5:
            monthName = "May";
            break;
         case 6:
            monthName = "June";
            break;
         case 7:
            monthName = "July";
            break;
         case 8:
            monthName = "August";
            break;
         case 9:
            monthName = "September";
            break;
         case 10:
            monthName = "October";
            break;
         case 11:
            monthName = "November";
            break;
         case 12:
            monthName = "December";
            break;
         default:
            monthName = "Bad Number";
            break;
      }

      System.out.println("Month number "+ month + " is " + monthName);
   }//end main method
}//end class

 

Drawback

The drawback to the switch statement is that it is easy to forget a break. Then the code just "falls through" and executes the following statements until a break is encountered. This can create bugs that are hard to find.
