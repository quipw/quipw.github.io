Decisions - if

 

Sequential Execution

Our programs so far have started at the beginning and run to the end - the same way every time. This is called sequential execution.

Flowchart of Sequential Program Flow

 

Selection

Our programs can be more useful if we can change the program flow - if we can check something and then decide what to do. This is called Selection. The if statement is used for selection in Java.

We make this kind of check all the time in real life. If it is sunny, I will go to the beach.

 

Syntax

The syntax of the if statement is

         if (boolean expression) statement;

A boolean expression is an expression that evaluates to either true or false.

(x > y) is a boolean expression. It is either true, or it is false.

 

We would write our decision about the beach like this:

if (sunny)
   go to the beach;

The expression (... ) in this case (sunny) is either true or false. true and false are called a Boolean values. boolean is a data type like int or double. A boolean expression can have only one of two values - true or false. In real life, the weather could be partly cloudy, but in Java the boolean expression, sunny, must be either true or false.

true and false are Java keywords.

 

Look at this code segment

boolean sunny = true;
System.out.println(sunny);

This will actually print the word "true".

 

Block Statement

But what if we wanted to do multiple things when an expression is true? We use what is called a block statement. We enclose several statements in curly braces { } and all of the statements are treated as a unit or block. They will all execute if the boolean expression is true. 

if (sunny)    //with a block statement 
{ 
   go to the beach; 
   put on sunscreen; 
   wade in the water; 
}
