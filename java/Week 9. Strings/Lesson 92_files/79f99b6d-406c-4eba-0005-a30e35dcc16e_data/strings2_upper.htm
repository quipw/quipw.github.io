toUpperCase and toLowerCase Methods

 

We have seen two string methods: charAt() and length(). There are many more methods in the String class. We will take a look at a few of them in this and the following sections. More are discussed in your text. For even more String methods, look At the String class in the API (Application Program Interface)

 

toUpperCase Method and toLowerCase

 

toUpperCase returns a string that is all upper case. It does not change the original String. If

String city = "Saratoga";

 

then

city.toUpperCase();

 

returns the String SARATOGA. The String city still references the the original object "Saratoga".

We need to assign the returned String to a reference variable if we want to use it later.

String upperCaseCity = city.toUpperCase();

 

We can also assign the new String to the existing reference city.

city = city.toUpperCase();

 

But now we no longer have a reference to the original string "Saratoga", although it still exists.

 

The toLowerCase method works similarly except that it returns a String that is all lowercase.

It is important to remember that these methods return a new string. They do not alter the original string. In fact, no string method can alter the original string. Strings are immutable (can't be changed).
