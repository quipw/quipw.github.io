Constants

 

One of the most important things you can do to make your code readable and self-documenting is to use named constants instead of using literal (the numerical values).

For instance, if you are using the number of days in a week in your program, don't simply code 7 each time you want the value. Make a constant. A constant is a variable that can't change. That sounds like an oxymoron (using contradictory terms together.) But we do it in Java.

You make a variable by specifying a data type and a name. You make the variable a constant by using the keyword final. final means the value can't be changed. By convention we will use all uppercase letters to name a constant.

final int DAYS_IN_WEEK = 7;

Also use this technique for values that are constant only in your program such as the width and height of a rectangle.

final int WIDTH = 60;
final int HEIGHT = 40;

The Java libraries contain many constants. One example is the value of pi in the Math class. We refer to it as Math.PI. You will encounter other Java constants in this class and in 27B.
